# 组织 AI 用例盘点表

适合谁：团队负责人、业务骨干、部门助理、项目经理，以及想系统找出 AI 落地场景的人。

你会得到什么：一张可以直接用于团队访谈和优先级排序的 AI 场景盘点表。它能帮你从“大家都觉得 AI 有用”走到“先做哪 3 个用例”。

建议用时：30 分钟准备，60 分钟访谈，30 分钟排序。

## 什么时候使用

当你需要回答下面这些问题时，用这张表：

- 我们团队到底有哪些任务适合用 AI？
- 哪些用例值得先做？
- 哪些想法听起来很酷，但现在不适合做？
- 试点应该从哪个部门、哪个流程、哪个角色开始？
- 如何向负责人说明为什么先做这个、不做那个？

## 使用原则

- 先盘点任务，不先盘点工具。
- 先找高频低风险场景，不先碰核心系统。
- 先做能验收的用例，不做只靠感觉判断的用例。
- 先让 AI 辅助人，不一开始追求无人化。
- 先完成一个小闭环，再谈组织级推广。

## 第一步：收集候选用例

找 3-8 个熟悉业务的人，每人列出 5 个“重复、耗时、容易出错、需要整理信息”的任务。

访谈时可以直接问：

```text
1. 你每周重复做 3 次以上的任务是什么？
2. 哪些任务最花时间，但价值不高？
3. 哪些任务需要整理大量资料、聊天记录、表格或文档？
4. 哪些任务经常因为信息不全、格式不统一而返工？
5. 哪些任务需要写初稿、改文案、做总结、生成表格？
6. 哪些任务可以先由 AI 做初稿，再由人检查？
7. 哪些任务绝对不能交给 AI 直接完成？
```

## 第二步：填写盘点表

复制下面表格。每一行代表一个候选 AI 用例。

| 编号 | 业务部门 | 使用角色 | 当前任务 | 当前痛点 | AI 可参与环节 | 输入材料 | 输出结果 | 频率 | 单次耗时 | 风险等级 | 是否可验证 | 优先级 | 下一步 |
| --- | --- | --- | --- | --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |
| 1 |  |  |  |  |  |  |  |  |  |  |  |  |  |
| 2 |  |  |  |  |  |  |  |  |  |  |  |  |  |
| 3 |  |  |  |  |  |  |  |  |  |  |  |  |  |

填写说明：

- 业务部门：例如市场部、销售部、客服部、行政部、财务部、交付部。
- 使用角色：真正会打开 AI 工具的人，不一定是负责人。
- 当前任务：写具体动作，例如“整理活动复盘”，不要写“提升市场效率”。
- 当前痛点：写耗时、返工、质量不稳定、资料分散、响应慢等具体问题。
- AI 可参与环节：写“生成初稿、提炼重点、对比差异、提取待办、检查格式”等。
- 输入材料：写清需要哪些资料，以及是否涉及客户、合同、财务、隐私数据。
- 输出结果：写 AI 最终交付什么，例如文档、表格、话术、摘要、检查清单。
- 频率：每天 / 每周 / 每月 / 不固定。
- 单次耗时：当前人工完成大约多久。
- 风险等级：低 / 中 / 高。
- 是否可验证：能不能用时间、质量、使用次数、业务结果判断效果。
- 优先级：A / B / C。
- 下一步：试点 / 补资料 / 暂缓 / 不做。

## 第三步：评分

给每个用例按 6 个维度打分，1 分最低，5 分最高。

| 维度 | 说明 | 1 分 | 5 分 |
| --- | --- | --- | --- |
| 高频程度 | 任务发生得是否频繁 | 很少发生 | 每天或每周多次发生 |
| 时间消耗 | 当前是否明显耗时 | 几分钟 | 每次 1 小时以上 |
| 资料清晰度 | 输入材料是否容易拿到 | 资料分散混乱 | 材料稳定、格式清楚 |
| AI 适配度 | AI 是否擅长参与 | 主要靠人判断和关系处理 | 适合总结、改写、提取、分类、生成初稿 |
| 风险可控度 | 出错后是否可控 | 影响客户、财务、法律或安全 | 只影响内部草稿，可人工复核 |
| 验证清晰度 | 是否能判断有没有效果 | 很难衡量 | 能用时间、质量、使用率判断 |

评分表：

| 编号 | 用例名称 | 高频程度 | 时间消耗 | 资料清晰度 | AI 适配度 | 风险可控度 | 验证清晰度 | 总分 | 建议 |
| --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |
| 1 |  |  |  |  |  |  |  |  |  |
| 2 |  |  |  |  |  |  |  |  |  |
| 3 |  |  |  |  |  |  |  |  |  |

建议判断：

- 24 分以上：优先试点。
- 18-23 分：可以进入候选池，先补资料或缩小范围。
- 17 分以下：暂缓，不作为第一批。
- 只要风险不可控，即使总分高，也不要进入第一批。

## 第四步：分成 4 类

盘点后，把用例放进下面 4 类。

| 类型 | 特征 | 处理方式 |
| --- | --- | --- |
| 立即试点 | 高频、耗时、低风险、可验证 | 放入 2 周试点 |
| 补资料后试点 | 有价值，但输入材料不稳定 | 先整理资料、模板、权限 |
| 观察等待 | 想法有价值，但依赖系统、数据或管理变化 | 记录下来，暂不启动 |
| 暂不适合 | 高风险、低频、难验证、责任不清 | 不进入本轮 AI 赋能 |

第一批只选 1-3 个“立即试点”用例。超过 3 个，团队很容易执行不下去。

## 适合第一批试点的用例

优先选择这些类型：

- 会议纪要整理。
- 活动复盘初稿。
- 周报、月报初稿。
- 客户沟通记录摘要。
- 培训资料提纲整理。
- 文案初稿和多版本改写。
- 竞品资料初筛。
- 招聘 JD 初稿和简历摘要。
- 内部知识库问答。
- 表格信息清洗和分类。

## 不适合作为第一批的用例

这些用例可以记录，但不建议第一批启动：

- 直接替代关键岗位决策。
- 自动给客户发送未经审核的内容。
- 自动判断客户归属、佣金、财务结算。
- 处理大量敏感个人信息且没有脱敏流程。
- 需要打通多个核心系统，但当前没有技术支持。
- 成功标准说不清，只能靠“感觉更智能”判断。
- 出错后会直接影响合同、付款、法律责任或客户权益。

## 从盘点表走向试点方案

选出 1-3 个优先用例后，每个用例继续填写《团队 AI 赋能方案画布》：

```text
用例盘点表回答：先做哪个场景。
团队 AI 赋能方案画布回答：这个场景具体怎么做。
```

不要跳过画布。盘点表只能说明“值得做”，画布才能说明“怎么做、谁负责、如何验收”。

## 最小可行版本

如果你没有时间做完整访谈，只问团队成员 3 个问题：

```text
1. 你每周最重复、最耗时的 3 个任务是什么？
2. 哪个任务可以先让 AI 做初稿，再由你检查？
3. 如果 AI 帮你做这件事，什么结果算成功？
```

把答案放进盘点表，再按评分规则排序。先从最高分、最低风险的那个任务开始。

## 完成后的下一步

1. 选出 1-3 个 A 类用例。
2. 为每个 A 类用例指定一个试点负责人。
3. 用《团队 AI 赋能方案画布》写清试点方案。
4. 准备统一提示词、输入材料样例和输出格式。
5. 用 2 周时间验证，不要一开始就扩到全员。
