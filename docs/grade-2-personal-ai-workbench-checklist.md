# 二年级：个人 AI 工作台搭建清单

适合谁：已经会使用 AI，但还没有把 AI 固定接入工作流程的人。

你会得到什么：一个围绕岗位任务搭建的个人 AI 工作台。它不是软件系统，而是一套你每天能复用的资料、提示词、输出格式和检查清单。

建议用时：90 分钟搭第一版，之后每周优化一次。

## 什么是个人 AI 工作台

个人 AI 工作台包含 5 个部分：

1. 高频任务：你最值得用 AI 改造的任务。
2. 输入材料：AI 每次需要哪些资料。
3. 提示词模板：你怎么稳定地向 AI 提要求。
4. 输出格式：AI 产出要长什么样。
5. 复核清单：哪些地方必须由你检查。

它解决的问题是：你不再每次从零开始问 AI，而是把一个任务变成可重复流程。

## 第一步：选一个高频任务

不要先搭一个万能工作台。先选一个任务。

优先选择符合这些条件的任务：

- 每周至少出现 2 次。
- 每次耗时 30 分钟以上。
- 需要整理资料、写初稿、做总结、改格式。
- 出错后可以人工修正，不会直接造成严重后果。
- 成功标准比较清楚。

候选任务示例：

- 写周报、月报。
- 整理会议纪要。
- 做客户沟通摘要。
- 写活动复盘。
- 做竞品资料整理。
- 改写对外文案。
- 整理学习笔记。
- 生成培训提纲。

填写：

```text
我要搭建工作台的任务是：

这个任务现在的痛点是：

我希望 AI 帮我节省的时间是：

这个任务的最终成果是：
```

## 第二步：整理输入材料

AI 输出不稳定，通常不是因为 AI 不行，而是输入材料不稳定。

为这个任务列出固定输入：

| 输入材料 | 来源 | 每次都需要吗 | 是否可上传 AI | 是否需要脱敏 |
| --- | --- | --- | --- | --- |
|  |  | 是 / 否 | 是 / 否 | 是 / 否 |
|  |  | 是 / 否 | 是 / 否 | 是 / 否 |
|  |  | 是 / 否 | 是 / 否 | 是 / 否 |

常见输入材料：

- 会议记录。
- 聊天记录。
- 客户资料。
- 表格数据。
- 产品说明。
- 历史案例。
- 参考文章。
- 负责人要求。

如果材料涉及客户、合同、财务、手机号、身份证、住址、内部价格，先脱敏再上传。

## 第三步：写固定提示词

提示词不需要复杂，但要稳定。

通用模板：

```text
你现在要帮我完成这个任务：{任务名称}

背景：
{任务背景}

输入材料：
{粘贴材料}

请输出：
{明确输出内容}

格式要求：
{表格/分点/报告/话术/清单}

质量要求：
1. 不要编造输入材料中没有的信息。
2. 信息缺失请标注“待补充”。
3. 重要判断请说明依据。
4. 输出要方便我直接修改和使用。

我的使用场景：
{发给谁/用于什么决策/用于什么流程}
```

追问模板：

```text
请基于上一版继续修改：
1. 删除不够具体的表达。
2. 把下一步动作写得更明确。
3. 标出需要人工确认的信息。
4. 用更适合{读者}的语气重写。
```

## 第四步：固定输出格式

AI 最终输出必须进入你的工作流程。格式不固定，就很难复用。

选择一种输出格式：

```text
我的输出格式：

标题：

背景：

关键信息：
- 
- 
- 

分析：
- 
- 
- 

下一步动作：
| 动作 | 负责人 | 截止时间 | 备注 |
| --- | --- | --- | --- |

需要人工确认：
- 
- 
```

如果是客户沟通类任务，加上：

```text
客户关注点：

客户可能的疑问：

建议回复话术：

不能直接发送的内容：
```

## 第五步：建立复核清单

AI 工作台必须有复核清单。没有复核，AI 输出很容易变成风险。

```text
复核清单

事实：
- 数字是否准确？
- 人名、公司名、时间是否正确？
- 是否有 AI 自行补充的信息？

语气：
- 是否适合读者？
- 是否有内部抱怨或流程解释？
- 是否有夸大承诺？

格式：
- 是否符合固定输出格式？
- 是否能直接复制到目标文档？

行动：
- 是否有下一步动作？
- 是否有负责人和截止时间？

风险：
- 是否包含隐私或敏感信息？
- 是否需要上级确认？
- 是否能直接对外发送？
```

## 第六步：放到固定位置

把工作台放在你每天容易打开的地方。

建议结构：

```text
个人 AI 工作台

1. 高频任务说明
2. 输入材料清单
3. 固定提示词
4. 输出格式
5. 复核清单
6. 优秀案例
7. 常见问题
```

可以放在飞书文档、Notion、语雀、Obsidian、石墨文档、腾讯文档，或本地 Markdown 文件里。

## 示例：周报 AI 工作台

```text
任务：生成个人周报初稿

输入材料：
- 本周完成事项
- 会议记录
- 项目进展
- 下周计划
- 遇到的问题

提示词：
请根据下面材料生成我的周报初稿。
读者是我的直属负责人。
请按“本周完成、关键进展、问题风险、下周计划、需要支持”输出。
不要夸大结果，不要写空话。
如果信息不足，请标注“待补充”。

输出格式：
1. 本周完成
2. 关键进展
3. 问题风险
4. 下周计划
5. 需要支持

复核重点：
- 是否有遗漏的重要事项
- 是否有不准确的项目状态
- 是否写成流水账
- 是否明确需要支持的事项
```

## 每周优化一次

每周五花 10 分钟更新工作台：

```text
本周哪条提示词最好用：

哪次 AI 输出最差：

差在哪里：

我要补充什么输入材料：

我要修改哪条提示词：

我要保存的优秀输出案例：
```

## 二年级通过标准

达到下面 5 条，就可以进入三年级：

- 已经为一个高频任务建立固定 AI 工作台。
- 有固定输入材料清单。
- 有至少 1 条稳定可复用提示词。
- 有固定输出格式。
- 有人工复核清单，并实际用过 2 次以上。
