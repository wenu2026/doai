# 二年级：岗位流程复盘模板

适合谁：想把 AI 接入岗位任务，但不知道从哪里下手的人。

你会得到什么：一份岗位流程复盘表。它能帮你把一个工作任务拆成输入、步骤、AI 可参与环节、人工复核点和改造后的新流程。

建议用时：45-60 分钟复盘一个任务。

## 什么时候使用

当你有一个任务“经常做、很耗时、容易返工”时，先用这个模板复盘，再决定是否用 AI 改造。

不要直接问“AI 能不能替我做这个岗位”。应该问：

```text
这个岗位里的哪一个任务，可以先让 AI 辅助一部分？
```

## 第一步：写清当前任务

```text
任务名称：

这个任务服务谁：

任务开始的触发条件：

任务结束的成果：

当前每周发生频率：

当前每次耗时：

当前最痛的地方：
```

示例：

```text
任务名称：整理客户拜访纪要
这个任务服务谁：销售主管和后续跟进销售
任务开始的触发条件：销售完成客户拜访
任务结束的成果：一份客户需求摘要和下一步跟进建议
当前每周发生频率：10-15 次
当前每次耗时：20-30 分钟
当前最痛的地方：记录格式不统一，主管很难快速判断客户重点
```

## 第二步：拆当前流程

把流程拆到 5-8 步。不要写太粗。

| 步骤 | 当前动作 | 输入材料 | 产出 | 耗时 | 常见问题 |
| --- | --- | --- | --- | --- | --- |
| 1 |  |  |  |  |  |
| 2 |  |  |  |  |  |
| 3 |  |  |  |  |  |

常见问题可以这样写：

- 信息缺失。
- 格式不统一。
- 需要反复改写。
- 需要从很多资料里找重点。
- 容易漏掉下一步动作。
- 需要人工判断，AI 不能直接决定。

## 第三步：判断 AI 适合参与哪一步

AI 更适合做这些事：

- 总结长文档。
- 提取关键信息。
- 改写文字。
- 生成初稿。
- 对比差异。
- 按格式整理表格。
- 检查遗漏。
- 生成备选方案。

AI 不适合直接做这些事：

- 最终业务决策。
- 敏感客户判断。
- 财务结算。
- 法律责任判断。
- 未经审核的对外发送。
- 涉及隐私数据的直接处理。

填写：

| 步骤 | AI 是否参与 | AI 做什么 | 人做什么 | 风险 |
| --- | --- | --- | --- | --- |
| 1 | 是 / 否 |  |  |  |
| 2 | 是 / 否 |  |  |  |
| 3 | 是 / 否 |  |  |  |

## 第四步：设计新流程

新流程要比旧流程更清楚，而不是只多一个 AI 工具。

```text
改造后的流程：

1. 人准备输入材料：
2. AI 生成第一版：
3. 人检查：
4. AI 修改：
5. 人确认：
6. 最终结果沉淀到：
```

输出格式：

```text
最终输出必须包含：
- 
- 
- 

不能出现：
- 
- 

必须人工确认：
- 
- 
```

## 第五步：写提示词

```text
你要帮我完成的任务是：{任务名称}

背景：
{这个任务服务谁、用于什么流程}

输入材料：
{粘贴材料}

请你先做：
1. 提取关键信息
2. 按指定格式生成初稿
3. 标出信息缺失或需要人工确认的地方

输出格式：
{粘贴固定格式}

注意：
- 不要编造输入材料里没有的信息。
- 不能确定的地方写“待确认”。
- 对外表达要自然、清楚，不要像内部流程说明。
```

## 完整示例：客户拜访纪要

```text
任务名称：整理客户拜访纪要

当前流程：
1. 销售手写或语音记录客户沟通。
2. 销售回到公司后整理成文字。
3. 销售主管阅读并判断客户重点。
4. 销售补充下一步跟进计划。

AI 可参与：
- 把原始记录整理成结构化摘要。
- 提取客户关注点、反对意见、下一步动作。
- 标出信息缺失。

人必须负责：
- 判断客户真实意向。
- 确认客户信息准确性。
- 决定下一步报价和跟进策略。

改造后流程：
1. 销售把拜访记录脱敏后放入模板。
2. AI 生成客户摘要和待确认问题。
3. 销售检查事实。
4. 主管查看统一格式摘要。
5. 销售确认下一步动作。

验收标准：
- 单次整理时间从 30 分钟降到 10 分钟以内。
- 主管能在 2 分钟内看懂客户重点。
- 连续 10 次纪要使用统一格式。
- 没有未经确认的客户判断直接进入对外话术。
```

## 常见错误

- 任务太大：例如“让 AI 改造销售”。应改为“让 AI 整理客户拜访纪要”。
- 输入不固定：每次给 AI 的材料都不一样，输出自然不稳定。
- 没有人复核：AI 初稿被直接对外使用。
- 只看节省时间：忽视事实错误、语气错误、隐私风险。
- 没有沉淀：每次都重新问，无法形成流程。

```text

2. 当前流程：
3. 当前痛点：
4. AI 可参与环节：
5. 人工复核点：
6. 改造后的新流程：
7. 固定提示词：
8. 固定输出格式：
9. 验收标准：
10. 试用一次后的结果：
```
